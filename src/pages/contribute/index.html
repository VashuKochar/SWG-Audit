Contributing to SWG Audit

Report issues, code changes, documentation

Report issues

Open an issue in the repository for bugs, issues, or suggestions. Include steps to reproduce and relevant screenshots or logs.

Submit code changes

  1. Fork the repository on GitHub.
  2. Create a branch with a descriptive name (e.g. fix-bug-123, add-new-feature).
  3. Implement changes in the appropriate files. Follow the project coding standards.
  4. Test changes for expected behaviour and no regressions.
  5. Submit a pull request to the main repository with a clear description of the changes.

Update documentation

Update missing or outdated documentation: README.md, code comments, or new documentation files.

Suggest features

Open an issue in the repository to discuss a new feature with the maintainers. Provide detail about the feature and benefits.

Code of conduct

Adhere to the project Code of Conduct.

Get help

Open an issue in the repository or contact the maintainers.
